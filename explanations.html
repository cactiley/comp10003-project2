Explanations

Text Formatting

Headings are used to create titles and subtitles across the webpage. The headings range from <h1> to <h4> to format and divide the sections depending on the content. For example, the <h1> heading 'Exploding Kittens' included in the <header> signifies that it is the main title displayed across all web pages. This creates emphasis and lets viewers know that the main content covered is about Exploding Kittens.

Paragraphs <p> are used to write descriptive text that goes below the heading. This includes content extending on the heading and explanations to provide further knowledge for viewers.

Media

Images were embedded through the <img> tag to provide graphics relating to the content of the web page. Images also enhance the appearance of the webpage to attract viewers. This goes for the GIF banner in the header consisting of a kitten with animated special effects. The <img> tag also contains two important attributes: <src> and <alt>. The <src> attribute retrieves and links to the location of the image. The <alt> attribute is an alternate description of the image, specifically for screen readers. This provides accessibility for users with vision impairment.

This website uses a combination of official graphics (images of cards and branding images from the Exploding Kittens official website), as well as hand-drawn images and collages. These complement text to better express points; for example, images of specific cards accompany their explanations, and diagrams of the setup follow the instructions.

Containers

Semantic containers divided webpages based on their function. The <header> container, containing <nav>, was kept uniform across pages. This allows fast, user-friendly navigation and a clear indication of the website's name and purpose. The <main> element of the webpage is divided into sections (using <section>) for clarity; topics and ideas are grouped visually.

Class selectors were used in <section> containers for CSS styling. The <div> container was also used to create components within sections visually distinctive. This allows large sections of text to be categorised for better readability.

Anchors

Anchors (using <a>) were added to direct users to external sites where references were provided. Use of links, rather than a text reference of another site's web address, allows users to go directly to another website, rather than having to manually reach the site.

Colors

The website's colour palette evokes a dull yet playful atmosphere. It contains a balance of monochromatic greys with a pop of red seen in the <nav>nav> bar. The color #5B6464 is achieved through the property background-color of the <body> and #591C14 through the same property of the <nav>. The background colour also differs from the subsections and cards to create a division between the background and the written text.

Selectors

Selectors are used to classify specific HTML elements and <div> classes for styling. This ensures that specific elements differ from one another to create a balanced layout. The .card class is an example of a selector containing properties such as display: flex and flex-direction: column, allowing multiples cards to stack on top of each other.

Fonts

Fonts helps to categorise different forms of text based on the content to display on the web page. Headings <h1> and <h2> uses the font-family: "Bebas Neue" to emphasise the main title across all web pages. Headings <h3> and <h4> uses the font-family: "Alegreya" as titles for subsections and cards.

Font families ensure a fallback option are available in case a user's browser does not support our chosen fonts. These fallback options are web-friendly to ensure our site has widespread accessibility.

Images

Images were styled into different sizes depending on its context varied across the web pages. Sizes are distinguished by the properties width to set the image in a specific length, and padding to create space around the image. This allows for a cleaner and consistent layout which appears more pleasant towards the viewers.

Images were all kept less than 100 KB. Compressing images allows the website to run smoothly. This was done by reducing the dimensions and converting file types; PNG images were converted to WEBP or JPEG to minimize file size.

Containers

Containers were styled differently, using class selectors where needed, to visually separate areas. This created distinct sections, primarily through colour choices and margins/padding.

The styling of containers varies and can be stored in class flex-container with display: flex to store elements in a single row. While class grid-container with display: grid helps to arrange elements in both a row and column simultaneously.

Objects

Objects were used to hold multiple values under a single variable. Using arrays of objects with let questions allows us to store all questions more efficiently instead of using an array for each question. Variable names are also written in camelCase (e.g. populatePage, validateQuiz) for clarity.

Because of the variety of questions for the quiz, each question contains a different variable type. This includes questions in a format of date, number, text, checkbox, and radio.

Objects

Functions are key to performing specific tasks to run and make changes to the code. For example, functions can be declared to check the answers of the quiz by performing function checkAnswers(), which returns outcomes based on the userAnswer. By using the if and else if statements, the outcome that returns will be true if (userAnswer == correctAnswer).

With the use of the if and else if statements, it can also return the output when the function is called. This goes for the function validateQuiz.

Loops

Loops help to execute a block of code to run multiple times. The use of the for ... of loop such as for (const question of questions) of the populatePage generates a button through document.createElement(button) for users to submit their quiz and receive feedback.
